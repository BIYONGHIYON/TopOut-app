# TopOut

**등반 시작부터 완등 후보 순간까지, 필요한 구간을 자동으로 찾아주는 클라이밍 촬영 앱**

iOS · Android

<sub>현재 개발 버전: 1.2</sub>

</div>

---

## 소개

**TopOut**은 실내 클라이머가 긴 촬영 영상에서 실제 등반 구간을 더 빠르게 찾고 저장할 수 있도록 돕는 모바일 앱입니다.

사용자가 촬영 구도와 매트 경계선을 설정하면 앱이 매트 주변의 영상 변화와 사람의 자세를 단계적으로 분석해 등반 시작과 종료 후보 시점을 찾습니다.

탑홀드 선택 기능을 사용하는 경우에는 사용자가 지정한 홀드와 신체 관절이 가장 가까워진 순간을 종료 후보로 사용합니다. 탑홀드를 선택하지 않은 경우에는 등반 중 신체가 가장 높은 위치에 도달한 순간을 종료 후보로 활용합니다.

정상적인 촬영 상황에서는 사용자가 직접 녹화를 종료할 때까지 하나의 원본 영상으로 계속 기록됩니다. 한 번의 연속 촬영 안에서 여러 번의 등반 시도를 각각 감지할 수 있으며, 촬영이 끝난 뒤 감지 구간을 확인하고 시작점과 종료점을 직접 미세 조정할 수 있습니다.

선택한 구간만 기기의 사진 또는 동영상 보관함으로 내보낼 수 있어, 긴 원본 영상을 다시 돌려보며 필요한 순간을 찾는 시간을 줄여줍니다.

## 주요 기능

### 매트 경계선 탐지 및 수동 보정

벽과 매트가 만나는 경계를 자동으로 찾고, 탐지 결과가 실제 환경과 다르면 사용자가 화면의 조절점을 드래그해 직접 수정할 수 있습니다.

### Rolling Frame Difference 기반 움직임 감지

촬영 시작 화면과 계속 비교하지 않고, 현재 분석 프레임을 **3개 분석 프레임 전**과 비교합니다.

이를 통해 촬영 시작 시 이미 존재하던 고정 노이즈, 조명 차이, 정적인 클라이밍 홀드 때문에 자세 분석이 불필요하게 계속 실행되는 현상을 줄입니다.

### Motion Detection → Pose Detection 단계형 분석

등반 시작 전부터 고비용 자세 모델을 계속 실행하지 않습니다.

1. 녹화 시작 후 매트 주변의 기준 영상을 보정합니다.
2. 매트 경계 주변에서 경량 Motion Detection을 수행합니다.
3. 의미 있는 움직임 후보가 확인된 경우에만 Pose Detection을 실행합니다.
4. 신뢰할 수 있는 사람 자세가 확인되면 등반 시작 여부를 판단합니다.

### 플랫폼별 온디바이스 자세 분석

모든 자세 분석은 사용자의 기기 안에서 수행됩니다.

- **iOS:** Apple Vision Human Body Pose
- **Android:** TensorFlow Lite 기반 MoveNet SinglePose Thunder FP16

촬영 영상이나 분석용 이미지 프레임은 TopOut 개발자 서버로 전송되지 않습니다.

### Android Smart Crop Pose Tracking

Android에서는 정적인 클라이밍 홀드를 사람으로 오인하지 않도록, 전체 화면에서 탐지된 자세가 실제 사람 형태를 갖추고 일정 시간 동안 움직였는지 먼저 확인합니다.

현재 흐름은 다음과 같습니다.

```text
전체 화면 Pose Detection
        ↓
3개 Pose 프레임 동안 관절 움직임 확인
        ↓
Smart Crop 활성화
        ↓
Crop 영역에서 유효한 Pose 2프레임 연속 확인
        ↓
등반 시작 판정 허용
        ↓
이전 프레임의 관절 위치로 다음 ROI 갱신
```

Smart Crop이 활성화된 뒤에는 전체 프레임을 다시 변환하지 않고, 이전 Pose에서 계산한 사람 ROI를 CameraX YUV 프레임에서 MoveNet 입력 크기로 직접 변환합니다. 이를 통해 추적 정확도를 유지하면서 중간 Bitmap 생성과 불필요한 이미지 변환을 줄입니다.

### MobileSAM 기반 탑홀드 선택

목표 홀드를 화면에서 탭해 선택할 수 있으며, MobileSAM이 선택 영역을 분할해 마스크와 외곽선으로 표시합니다.

- **iOS:** Core ML MobileSAM
- **Android:** ONNX Runtime MobileSAM

### 자동 등반 구간 감지

매트 경계 통과 여부, 사람 자세, 관절 위치 변화를 바탕으로 등반 시작과 종료 후보 시점을 계산합니다.

- 탑홀드 선택 ON: 탑홀드와 신체 관절이 가장 가까웠던 시점
- 탑홀드 선택 OFF: 등반 중 신체가 가장 높았던 시점

감지 결과는 자동 편집을 위한 후보 구간이며, 사용자가 결과 화면에서 직접 수정할 수 있습니다.

### 다중 시도 감지

완등 후보 시점이 확인되어도 녹화가 자동으로 종료되지 않습니다.

사용자가 종료 버튼을 누를 때까지 하나의 원본 영상을 계속 기록하며, 한 번의 촬영 안에서 여러 등반 시도를 각각 구분해 결과 화면에 제공합니다.

### 자동 선택 및 정밀 트리밍

촬영 종료 후 감지된 등반 구간을 자동으로 제안합니다.

- 감지된 여러 시도 간 전환
- 시작점·종료점 수동 조정
- 느린 드래그를 통한 정밀 조정
- 탑홀드 외곽선 포함 여부 선택
- 선택 구간만 저장

### 녹화 중 실시간 화면 OFF

기본 설정에서는 녹화 중 실시간 카메라 화면을 표시하지 않습니다.

카메라 센서, 영상 녹화, Motion Detection 및 Pose Detection은 계속 동작하지만 화면용 프리뷰 렌더링을 중단해 장시간 촬영 시 화면 표시로 인한 배터리 소모와 발열을 줄입니다.

사용자는 설정에서 녹화 중 실시간 화면을 다시 켤 수 있습니다.

### 안전한 녹화 종료

다음 상황에서는 현재까지 촬영된 영상이 손상되지 않도록 녹화를 안전하게 마무리합니다.

- 앱이 백그라운드로 이동한 경우
- 전화 수신 또는 다른 앱의 카메라·마이크 사용으로 세션이 중단된 경우
- 카메라 세션 오류가 발생한 경우
- 남은 저장공간이 안전 기준보다 부족한 경우

## 작동 흐름

1. 촬영 해상도, 프레임률, 화면 비율과 자동 탐지 옵션을 설정합니다.
2. 벽, 매트, 클라이머가 모두 보이도록 촬영 구도를 맞춥니다.
3. 자동으로 감지된 매트 경계선을 확인하고 필요한 경우 직접 보정합니다.
4. 탑홀드 선택 기능을 사용하는 경우 완등 목표 홀드를 지정합니다.
5. 녹화를 시작하면 앱이 매트 주변의 기준 영상을 보정합니다.
6. 이후 현재 분석 프레임과 3개 분석 프레임 전을 비교해 경계 주변의 움직임을 감지합니다.
7. 움직임 후보가 확인되면 기기 내 Pose Detection을 실행해 사람과 등반 시작 여부를 확인합니다.
8. 등반 중에는 자세와 관절 위치를 분석해 종료 후보 순간을 계속 갱신합니다.
9. 클라이머가 다시 매트로 내려오거나 Pose가 일정 시간 이상 사라지면 현재 시도를 마무리합니다.
10. 한 번의 촬영에서 여러 시도가 발생하면 각 등반 구간을 별도로 기록합니다.
11. 녹화 종료 후 자동으로 제안된 구간을 확인하고 시작점과 종료점을 직접 조정합니다.
12. 원하는 구간만 기기의 사진 또는 동영상 보관함에 저장합니다.

## Android 자세 분석 구조

Android에서는 전체 화면 탐지와 추적 단계의 처리 방식을 분리합니다.

### 1. 초기 탐지

```text
CameraX YUV 프레임
        ↓
전체 화면을 긴 변 최대 512px로 변환
        ↓
Bilinear resize
        ↓
MoveNet Thunder 256×256 입력
```

### 2. 사람 형태 및 움직임 검증

얼굴 관절을 제외한 주요 관절을 사용해 사람 형태를 확인하고, 3개의 Pose 분석 프레임에서 공통 관절의 이동량을 비교합니다.

정적인 홀드가 사람처럼 검출된 경우 Smart Crop이 활성화되지 않도록, 사람 크기에 비례한 실제 관절 움직임이 확인되어야 합니다.

### 3. Smart Crop 활성화 및 확인

Smart Crop이 활성화된 뒤 Crop 영역에서 유효한 사람 Pose가 **2프레임 연속** 확인되어야 등반 시작 판정을 허용합니다.

### 4. ROI 직접 변환 및 추적

```text
이전 Pose의 골반 중심과 관절 범위
        ↓
다음 프레임의 정사각형 ROI 계산
        ↓
CameraX YUV에서 ROI만 직접 샘플링
        ↓
256×256 RGB 입력 생성
        ↓
MoveNet Thunder 추론
        ↓
전체 화면 좌표로 복원
        ↓
다음 ROI 갱신
```

Smart Crop 이후에는 512px 전체 RGB Bitmap을 만들지 않아 이미지 변환량과 메모리 할당을 줄입니다.

## 공통 분석 기준

| 항목 | 현재 기준 |
|---|---:|
| Pose 분석 간격 | 0.5초 |
| 자동 구간 시작 전 여백 | 3초 |
| 종료 후보 이후 여백 | 3초 |
| Pose 미검출 허용 시간 | 6초 |
| 짧은 원본 감지 구간 기준 | 2초 미만 |
| 인접 구간 병합 간격 | 3초 이내 |
| 최대 감지 구간 수 | 최근 10개 |

짧은 원본 감지 구간이 2초 미만이면 인접 구간과의 간격을 확인해 병합하거나 제외합니다. 최종 저장 구간에는 시작 전·종료 후 여백이 별도로 적용됩니다.

## 기본 설정

| 항목 | 기본값 |
|---|---|
| 해상도 | FHD |
| 프레임률 | 60fps |
| 화면 비율 | 9:16 |
| 자동 구간 탐지 | ON |
| 탑홀드 마스킹 | OFF |
| 도움말 | ON |

지원 후보:

- 해상도: FHD, 4K
- 프레임률: 30fps, 60fps
- 화면 비율: 9:16, 4:5, 3:4, 1:1
- 세로·가로 촬영 지원

실제 선택 가능한 해상도와 프레임 조합은 기기의 후면 카메라 성능에 따라 달라집니다. 저장된 설정이 현재 기기에서 지원되지 않으면 가능한 조합으로 자동 보정됩니다.

## 분석 구조

```text
매트 경계 주변 CV 변화 감지
        ↓
현재 프레임과 3개 분석 프레임 전 비교
        ↓
움직임 후보 확인
        ↓
기기 내 사람 자세 분석
        ↓
사람 형태 및 움직임 검증
        ↓
매트 경계 통과 여부 확인
        ↓
등반 시작 구간 기록
        ↓
탑홀드 근접 또는 최고 도달 순간 계산
        ↓
종료 후보 구간 기록
        ↓
사용자 검토 및 정밀 트리밍
```

영상 분석은 실시간 녹화 영상에서 수행되지만, 감지 결과는 최종 판단이 아닌 **편집 후보 구간을 제안하기 위한 보조 정보**로 사용됩니다.

## 개인정보 및 영상 처리

- TopOut은 회원가입 없이 사용할 수 있습니다.
- 이름, 이메일 주소, 위치정보, 기기 식별자 등의 개인정보를 수집하지 않습니다.
- 촬영한 원본 영상과 분석용 이미지 프레임을 TopOut 개발자 서버로 업로드하지 않습니다.
- 움직임 감지, 자세 추정, 탑홀드 분할 등의 영상 분석은 사용자의 기기 내에서 수행됩니다.
- 사용자가 저장을 선택한 결과 영상만 기기의 사진 또는 동영상 보관함으로 내보냅니다.
- 현재 버전에는 광고 및 외부 사용자 행동 분석 도구가 포함되어 있지 않습니다.

자세한 내용은 아래 개인정보 처리방침에서 확인할 수 있습니다.

[TopOut 개인정보 처리방침](https://topoutprivacy.vercel.app/)
